# Обновленная авторизация GDT Update Module

## Изменения в системе авторизации

В версии 2.0 система авторизации была упрощена и улучшена:

### Что изменилось:
- **Убран параметр `client_id`** - больше не используется для авторизации
- **Оставлен только `api_key`** - единственный параметр для аутентификации
- **Упрощены настройки** - в админ панели убрано поле "Client ID"

### Серверная авторизация
Сервер проверяет авторизацию следующим образом:

1. **API ключ** - проверяется в параметре `api_key` POST запроса
2. **IP адрес** - проверяется список разрешенных IP (если настроен)

### Настройка клиента (модуль обновлений)

В админ панели OpenCart перейдите в **Extensions > Modules > GDT Updater** и настройте:

1. **URL сервера обновлений** - адрес вашего сервера
2. **API-ключ** - секретный ключ для доступа к серверу
3. **Статус** - включить/выключить модуль

### Настройка сервера

В админ панели OpenCart на сервере перейдите в **Extensions > Modules > GDT Update Server** и настройте:

1. **API ключ** - тот же ключ, что и у клиентов
2. **Разрешенные IP** - список IP адресов клиентов (или * для всех)
3. **Включить логирование** - для отладки

### Пример API запроса

```php
// Проверка обновлений
POST /index.php?route=gdt_update_server/check
{
    "api_key": "your_secret_api_key_123",
    "code": "module_code",
    "version": "1.0.0"
}

// Загрузка модуля  
POST /index.php?route=gdt_update_server/download
{
    "api_key": "your_secret_api_key_123",
    "code": "module_code"
}

// Список модулей
GET /index.php?route=gdt_update_server/modules?api_key=your_secret_api_key_123
```

### Тестирование

Используйте файл `test_auth.php` для проверки правильности настройки авторизации:

```bash
php test_auth.php
```

### Миграция с предыдущей версии

1. Обновите файлы модуля на сервере и клиентах
2. В настройках клиентов уберите значение "Client ID" (поле больше не отображается)
3. Убедитесь, что API ключ одинаковый на сервере и всех клиентах
4. Протестируйте соединение с помощью `test_auth.php`

### Безопасность

- Используйте сложный API ключ (минимум 20 символов)
- Ограничьте доступ по IP адресам если возможно
- Включите HTTPS для защиты передачи API ключа
- Регулярно меняйте API ключ

### Решение проблем

**Ошибка "Unauthorized":**
- Проверьте правильность API ключа
- Убедитесь, что IP клиента в списке разрешенных

**Ошибка "Service unavailable":**  
- Проверьте, что сервер обновлений включен
- Убедитесь, что модуль сервера установлен

**Ошибка подключения:**
- Проверьте правильность URL сервера
- Убедитесь в доступности сервера по сети
