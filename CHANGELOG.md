# Changelog для GDT Module Manager

## Версия 2024-10-18

### Основные изменения

Модифицирован класс `Manager` для поддержки получения установленных модулей из всех источников OpenCart 3.0.3.7:

### Новая функциональность

1. **Мультиисточниковое получение модулей**
   - Модули из собственной таблицы `gdt_modules`
   - Модули из таблицы модификаторов OpenCart (`oc_modification`)  
   - Модули из системных файлов `.ocmod.xml` в папке `/system/`

2. **Обновленные методы:**

   - `getInstalledModules()` - теперь собирает модули из всех источников
   - `getModuleByCode($code)` - ищет модуль во всех источниках
   - `uninstall($module_code)` - поддерживает удаление модулей разных типов

3. **Новые приватные методы:**

   - `getModulesFromOpenCartModifications()` - получение модулей из таблицы модификаторов
   - `getModulesFromSystemFiles()` - получение модулей из системных файлов
   - `parseOcmodFile($file_path)` - парсинг XML файлов модификаторов
   - `getModuleFromOpenCartModifications($code)` - поиск конкретного модуля в модификаторах
   - `getModuleFromSystemFiles($code)` - поиск конкретного модуля в системных файлах
   - `removeModuleBySource($module, $module_code)` - удаление модуля в зависимости от источника
   - `removeOpenCartModification($module)` - деактивация модификатора OpenCart
   - `removeSystemFile($module)` - удаление системного файла с созданием резервной копии

### Поддерживаемые источники модулей

1. **GDT Modules Database** (`gdt_modules` table)
   - Собственная система управления модулями
   - Хранение в JSON формате в БД

2. **OpenCart Modifications** (`oc_modification` table)  
   - Стандартные модификаторы OpenCart
   - Только активные модификаторы (status = 1)

3. **System Files** (`.ocmod.xml` files in `/system/`)
   - Модификаторы в виде XML файлов
   - Автоматический парсинг метаданных

### Дедупликация

Система автоматически удаляет дубликаты модулей по коду (`code`), приоритет отдается:
1. GDT Modules Database
2. OpenCart Modifications  
3. System Files

### Безопасность при удалении

- Модификаторы OpenCart деактивируются, а не удаляются полностью
- Системные файлы копируются в резервную копию перед удалением (с timestamp)
- Подробное логирование всех операций

### Совместимость

- Полностью совместимо с существующим API
- Поддержка OpenCart 3.0.3.7
- Обратная совместимость с предыдущими версиями кода
