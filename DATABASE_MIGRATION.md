# Миграция на хранение в базе данных

## Обзор изменений

Модуль GDT Update Server был обновлен для хранения информации о модулях в базе данных вместо JSON файлов. Это обеспечивает:

- Лучшую производительность при работе с большим количеством модулей
- Расширенную функциональность поиска и фильтрации
- Интеграцию с каталогом модулей OpenCart
- Статистику скачиваний и рейтинги
- Централизованное управление

## Новые возможности

### 1. Структура базы данных

Создана таблица `gdt_server_modules` со следующими полями:

- **module_id** - Уникальный идентификатор
- **code** - Код модуля (уникальный)
- **name** - Название модуля
- **description** - Описание
- **version** - Версия
- **author** - Автор
- **author_url** - Сайт автора
- **category** - Категория модуля
- **opencart_version** - Совместимость с версией OpenCart
- **dependencies** - Зависимости (JSON)
- **archive_structure** - Структура архива (opencart/direct)
- **file_path** - Путь к файлу модуля
- **file_size** - Размер файла
- **file_hash** - MD5 хеш файла
- **image** - Иконка модуля
- **demo_url** - Ссылка на демо
- **documentation_url** - Ссылка на документацию
- **support_url** - Ссылка на поддержку
- **price** - Цена модуля
- **downloads** - Количество скачиваний
- **rating** - Рейтинг
- **reviews** - Количество отзывов
- **status** - Статус (активен/неактивен)
- **featured** - Рекомендуемый модуль
- **sort_order** - Порядок сортировки
- **date_added** - Дата добавления
- **date_modified** - Дата изменения

### 2. API для каталога модулей

#### Получение списка модулей

```
GET /index.php?route=gdt_update_server/modules&api_key=YOUR_API_KEY
```

Параметры:
- `category` - Фильтр по категории
- `featured` - Только рекомендуемые модули (1/0)
- `search` - Поиск по названию
- `limit` - Количество результатов (по умолчанию 20)
- `page` - Номер страницы

Ответ:
```json
{
  "success": true,
  "modules": [
    {
      "code": "test_module",
      "name": "Test Module",
      "description": "Description",
      "version": "1.0.0",
      "author": "Author",
      "category": "module",
      "price": 0.0000,
      "downloads": 15,
      "rating": 4.5,
      "featured": false,
      "date_added": "2024-01-01 12:00:00"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 20
}
```

#### Проверка обновлений

```
POST /index.php?route=gdt_update_server/check
```

Данные:
```json
{
  "code": "module_code",
  "version": "1.0.0",
  "api_key": "YOUR_API_KEY"
}
```

Ответ (если доступно обновление):
```json
{
  "status": "update_available",
  "version": "1.1.0",
  "name": "Module Name",
  "description": "Updated description",
  "author": "Author",
  "author_url": "https://example.com",
  "category": "module",
  "opencart_version": "3.0.x",
  "dependencies": [],
  "file_size": 1024000,
  "date_modified": "2024-01-01 12:00:00"
}
```

### 3. Расширенные возможности управления

- Статистика скачиваний
- Рейтинговая система
- Рекомендуемые модули
- Категоризация
- Поиск и фильтрация

## Миграция данных

### Автоматическая миграция

1. Скопируйте файл `tools/migrate_modules_to_db.php` в корень OpenCart
2. Раскомментируйте код в конце файла
3. Запустите скрипт из браузера или командной строки
4. Проверьте результаты миграции

### Ручная миграция

1. Установите обновленный модуль
2. Таблица создастся автоматически при установке
3. Загрузите модули заново через админку

## Обратная совместимость

- API endpoints остались прежними
- Формат ответов совместим с предыдущими версиями
- Добавлены новые поля в ответах

## Обновление клиентов

Клиенты должны обновить свои модули для использования новых возможностей:

1. Поддержка новых полей в ответах API
2. Использование расширенной информации о модулях
3. Интеграция с системой рейтингов

## Конфигурация

После обновления проверьте настройки модуля:

1. Статус модуля
2. API ключ
3. Разрешенные IP адреса
4. Логирование

## Поддержка

При возникновении проблем:

1. Проверьте логи OpenCart
2. Убедитесь, что таблица создана корректно
3. Проверьте права доступа к файлам модулей
4. Убедитесь в корректности API ключей
